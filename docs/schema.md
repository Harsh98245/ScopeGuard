# Database schema reference

Authoritative description of every table and column. Generated by hand from `prisma/schema.prisma` so it can carry richer commentary than the schema file itself. **When you change the Prisma schema, update this file in the same PR.**

All tables live in the `public` schema and have RLS enabled (see `prisma/migrations/*_enable_rls/migration.sql`).

---

## `users`

ScopeGuard customer. The row id mirrors the Supabase `auth.users.id` so RLS can use `auth.uid()` directly.

| Column                | Type                  | Constraints                          | Description                                                                            |
| --------------------- | --------------------- | ------------------------------------ | -------------------------------------------------------------------------------------- |
| `id`                    | `uuid`            | PK, default `gen_random_uuid()`     | Equals `auth.users.id`.                                                                |
| `email`                 | `text`            | UNIQUE, NOT NULL                    | Login email. Synced from Supabase auth.                                                |
| `stripeCustomerId`      | `text`            | UNIQUE, nullable                    | Created on first checkout.                                                             |
| `stripeSubscriptionId`  | `text`            | UNIQUE, nullable                    | Active Stripe subscription. Null on FREE or after cancellation.                        |
| `stripePriceId`         | `text`            | nullable                            | Currently-billed Stripe price ID; drives `tierFromPriceId` resolution.                 |
| `subscriptionStatus`    | `text`            | nullable                            | Stripe sub status: `active|trialing|past_due|canceled|unpaid|incomplete[_expired]|paused`. |
| `currentPeriodEnd`      | `timestamp(3)`    | nullable                            | End of current billing period. Surfaced as "Renews on" / "Cancels on".                 |
| `planTier`              | `PlanTier`        | NOT NULL, default `FREE`            | Active subscription tier; mutated only by Stripe webhook.                              |
| `inboundEmailAlias`     | `text`            | UNIQUE, NOT NULL                    | Per-user forwarding alias, e.g. `jane-abc123@inbound.scopeguard.app`.                  |
| `timezone`              | `text`            | NOT NULL, default `America/Toronto` | IANA tz used for date formatting.                                                       |
| `jurisdiction`          | `Jurisdiction`    | NOT NULL, default `US`              | Tax jurisdiction; drives `lib/finances/tax/`.                                          |
| `createdAt`             | `timestamp(3)`    | NOT NULL, default `now()`           |                                                                                        |
| `updatedAt`             | `timestamp(3)`    | NOT NULL                            | Auto-managed by Prisma `@updatedAt`.                                                   |

Indexes: `users_planTier_idx`. Subscription columns are mutated EXCLUSIVELY by `app/api/webhooks/stripe/route.ts` — application code reads them but never writes them directly.

## `projects`

| Column        | Type                | Constraints                  | Description                                                  |
| ------------- | ------------------- | ---------------------------- | ------------------------------------------------------------ |
| `id`          | `uuid`              | PK                           |                                                              |

| `hourlyRate`  | `decimal(10,2)`     | nullable                     | Null for fixed-fee.                                          |

Indexes: `(userId, status)`, `(userId, clientEmail)`.

## `transactions`

Unified financial transaction synced from any `IntegrationSource`. The `(source, externalId)` unique constraint guarantees idempotent syncs.

| Column          | Type                  | Constraints                              | Description                                                |
| --------------- | --------------------- | ---------------------------------------- | ---------------------------------------------------------- |
| `id`            | `uuid`                | PK                                       |                                                            |
| `userId`        | `uuid`                | FK → users.id, CASCADE                   |                                                            |
| `source`        | `IntegrationSource`   | NOT NULL                                 |                                                            |
| `externalId`    | `text`                | NOT NULL, UNIQUE with `source`           |                                                            |
| `type`          | `TransactionType`     | NOT NULL                                 | INCOME or EXPENSE.                                         |
| `amount`        | `decimal(12,2)`       | NOT NULL                                 | Always positive; `type` carries direction.                 |
| `currency`      | `text`                | NOT NULL                                 | ISO 4217.                                                  |
| `description`   | `text`                | nullable                                 |                                                            |
| `category`      | `text`                | nullable                                 | AI-assigned (Software, Travel, etc.).                      |
| `taxDeductible` | `boolean`             | NOT NULL, default false                  |                                                            |
| `receiptKey`    | `text`                | nullable                                 | Object key in `receipts` bucket.                           |
| `occurredAt`    | `timestamp(3)`        | NOT NULL                                 | When it happened, not when synced.                         |
| `createdAt`     | `timestamp(3)`        | NOT NULL                                 |                                                            |

Indexes: `(userId, occurredAt DESC)`, `(userId, type, occurredAt)`.

## `integrations`

OAuth/API-key linkage. One row per `(userId, source)`. **Tokens are AES-256-GCM ciphertext — never log.**

| Column           | Type                 | Constraints                        | Description                                                   |
| ---------------- | -------------------- | ---------------------------------- | ------------------------------------------------------------- |
| `id`             | `uuid`               | PK                                 |                                                               |
| `userId`         | `uuid`               | FK → users.id, CASCADE             |                                                               |
| `source`         | `IntegrationSource`  | NOT NULL, UNIQUE with `userId`     |                                                               |
| `accessToken`    | `text`               | NOT NULL                           | `iv:tag:ciphertext` hex bundle.                               |
| `refreshToken`   | `text`               | nullable                           | Same encoding.                                                |
| `tokenExpiresAt` | `timestamp(3)`       | nullable                           |                                                               |
| `lastSyncedAt`   | `timestamp(3)`       | nullable                           |                                                               |
| `isActive`       | `boolean`            | NOT NULL, default true             | Soft-disable rather than delete for audit retention.          |
| `metadata`       | `jsonb`              | nullable                           | Per-source schema documented below.                           |
| `createdAt`      | `timestamp(3)`       | NOT NULL                           |                                                               |

### Per-source `metadata` shapes

| `source`  | Shape                                                                |
| --------- | -------------------------------------------------------------------- |
| `STRIPE`  | `{ "accountId": string }` — Stripe Connect account ID.               |
| `PAYPAL`  | `{ "merchantId": string }`                                           |
| `PLAID`   | `{ "itemId": string, "institutionName": string }`                    |
| _others_  | Documented when wired.                                               |
